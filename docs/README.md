# Docs 生成計畫

本文件重新規劃 `docs/` 結構，僅保留一份報告式內容，並集中在：

- 研究動機
- 研究方法
- 分析結果
- 與一般預期不同的觀察
- 結論與建議

## 目前狀態

已清除 `docs/` 中原本的章節檔案，僅保留 `docs/README.md` 作為本次重整說明。後續可以根據以下規劃，建立五個主要報告章節檔案。

## 新報告章節規劃

1. `01_RESEARCH_MOTIVATION.md`
   - 研究動機與背景
   - 為何要分析 2021 年 Stack Overflow
   - 這份研究對技術問答社群的意義

2. `02_RESEARCH_METHODS.md`
   - 研究方法與分析設計
   - 使用的資料來源與分析流程
   - 這裡不必寫技術細節，重點在於「怎麼做」與「為何這樣做」

3. `03_RESULTS.md`
   - 各分析面向的主要結果摘要
   - 先以文字描述發現，後續可再嵌入圖片
   - 強調觀察而非程式實作

4. `04_UNEXPECTED_OBSERVATIONS.md`
   - 與一般預期不同的觀察
   - 這些結果為何令人意外或值得進一步思考
   - 對 Stack Overflow 社群生態的可能啟示

5. `05_CONCLUSIONS_AND_RECOMMENDATIONS.md`
   - 研究結論
   - 對未來研究、實務或政策的建議

## 撰寫原則

本次 `docs/` 目的不是開發手冊，而是一份研究報告。寫作時請注意：

- 以「研究故事」為主，不要細數程式碼或技術實作。
- 強調研究動機、方法設計、發現與洞察。
- 先把文字內容完成，圖片可後續補入。
- 觀察要有「與常見直覺不同」的角度。

## 生成 PDF 的原則

完成上述五個章節後，可將它們依序合併成報告：

1. `01_RESEARCH_MOTIVATION.md`
2. `02_RESEARCH_METHODS.md`
3. `03_RESULTS.md`
4. `04_UNEXPECTED_OBSERVATIONS.md`
5. `05_CONCLUSIONS_AND_RECOMMENDATIONS.md`

若需要，保留 `docs/metadata.yaml` 作為 Pandoc 生成設定，但本次並不需要額外建立多餘的章節檔案。

## 後續作業建議

1. 建立並填寫上述五個章節檔案。
2. 先把研究內容和觀察寫出來，後續再補圖片與數據。
3. 若有需要，可將其他研究資料的內容拆解成這五個章節。
